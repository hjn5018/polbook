# 🚀 미니 게시판 (Spring-React) 튜토리얼: 사용자 시나리오 (User Scenarios)

이 문서는 본격적인 Polbook 프로젝트 개발 전, 프론트엔드와 백엔드 통신을 연습하기 위해 구축하는 **"초소형 튜토리얼 게시판"**의 서비스 이용 흐름을 묘사합니다.

## 🌱 초보 개발자(관리자이자 유일한 사용자) 시나리오

**목표: 텅 빈 React 도화지에 책 제목과 가격을 입력하여 백엔드(Spring)를 거쳐 DB(MySQL)에 저장하고, 즉시 화면에서 그 결과를 확인하고 싶다.**

1.  **화면 접속 및 빈 목록 확인**
    *   React 서버(`http://localhost:3000`)에 접속합니다.
    *   상단에는 "새로운 책 등록하기" 폼(제목, 가격 입력칸)이 있고, 하단 "등록된 책 목록" 영역은 아직 DB가 비어있기 때문에 빈 화면으로 표시됩니다.
    *   (이때 백엔드 `http://localhost:8080/api/books`에서 GET 요청으로 빈 리스트 `[]`를 성공적으로 받아옵니다.)
2.  **데이터 입력 및 등록(POST) 액션**
    *   '책 제목' 칸에 `스프링 부트와 AWS로 혼자 구현하는 웹 서비스`를 입력합니다.
    *   '가격' 칸에 `18000`을 입력합니다.
    *   **[백엔드로 전송!]** 버튼을 클릭합니다.
    *   React의 `fetch()` 함수가 백엔드로 JSON 데이터를 던지며, Spring Boot의 Controller ➔ Service ➔ Repository를 타고 MySQL DB에 INSERT 쿼리가 실행됩니다.
3.  **성공 알림 및 화면 즉시 갱신(Refetch)**
    *   모든 과정이 끝나면 화면 상단에 "책이 성공적으로 등록되었습니다!"라는 알림창(alert)이 뜹니다.
    *   알림창 확인을 누르자마자, 하단 "등록된 책 목록" 영역에 방금 입력한 책 데이터가 `- 스프링 부트와 AWS로 혼자 구현하는 웹 서비스 / 18000원` 형태로 리렌더링되어 최신화됩니다.
4.  **반복 테스트**
    *   이 과정을 통해 제목과 숫자 값을 바꿔가며 2~3권의 책을 연달아 등록해 보고, DB 테이블(`Book`)에 데이터가 차곡차곡 쌓이는지(1, 2, 3...) 눈으로 확인하며 통신 성공의 기쁨을 누릅니다!
