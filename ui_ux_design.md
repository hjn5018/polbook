# 🎨 Polbook UI/UX 화면 설계서 (Wireframe 명세)

본 문서는 Polbook(교내 중고 책 거래 서비스)의 주요 6개 화면에 대한 레이아웃 구조와 사용자 상호작용(UX) 요소들을 정의합니다. 프론트엔드 컴포넌트를 분리하고 API를 설계하는 기준이 됩니다.

기본 모바일(앱/모바일 웹) 화면 비율을 기준으로 설계하며, 데스크톱에서는 양옆 공백(반응형 max-width)을 두어 모바일 뷰포트 경험을 유지합니다.

---

## 1. 🔐 로그인 / 회원가입 화면 (Auth)

학교 이메일(.kopo.ac.kr) 인증을 강제하여 외부인 접근을 차단하는 것이 핵심입니다.

### [로그인 탭]
*   **로고 & 타이틀:** 화면 상단 중앙에 'Polbook' 로고 배치
*   **입력 폼:**
    *   `학번` (텍스트 입력창)
    *   `비밀번호` (비밀번호 입력창, 숨김/표시 토글 아이콘)
*   **액션 버튼:**
    *   [로그인] (강조된 Primary 버튼)
*   **보조 링크:**
    *   비밀번호 찾기 | 회원가입 하러가기

### [회원가입 탭]
*   **1단계 (이메일 인증):**
    *   입력: `학번` + `@office.kopo.ac.kr` (도메인 고정 노출)
    *   버튼: [인증번호 발송]
    *   입력: `인증번호 6자리` 
    *   버튼: [인증 확인]
*   **2단계 (정보 입력):** (인증 완료 시 활성화)
    *   입력: `비밀번호`, `비밀번호 확인`, `닉네임`
    *   버튼: [Polbook 가입 완료]

### 💡 [비밀번호 찾기 모달 (UI 재사용)]
*   기존 회원가입의 '이메일 인증' 컴포넌트를 그대로 재사용하여 바텀 시트나 모달로 띄웁니다.
*   학번 입력 → 이메일로 인증번호 수신 → 인증 성공 시 **[새 비밀번호 설정]** 폼 표출

## 2. 🏠 메인 홈 화면 (Main Feed)

사용자가 로그인 후 가장 먼저 보는 화면으로, 판매 중인 책 리스트와 검색 및 필터를 제공합니다.

*   **[Top Bar (Header)]**
    *   좌측: 로고 (Polbook)
    *   우측: 🔍 검색 아이콘 (클릭 시 전체화면 검색창 또는 펼침창 표출), 🔔 알림 아이콘(배지 표시)
*   **[필터 & 카테고리 (Sticky Header)]**
    *   가로 스크롤 칩(Chip) 버튼들: `전체`, `전공 서적`, `교양 서적`, `자격증 서적`
    *   드롭다운 필터: `최신순/가격순`, `판매중만 보기` 토글
*   **[책 목록 피드 (List / Grid View)]**
    *   **도서 카드 (Book Card):**
        *   왼측/상단: 대표 도서 이미지 (정사각형 썸네일)
        *   우측/하단: 
            *   책 제목 (최대 2줄, 이후 말줄임)
            *   거래 장소 (예: "본관 입구 앞") & 등록 시간 (예: "2시간 전")
            *   가격 (굵은 텍스트)
            *   우측 하단 아이콘: 찜(❤️) 수, 채팅(💬) 수
            *   상태 라벨: [예약중], [거래완료] (회색/반투명 처리 됨)
*   **[Floating Action Button (FAB)]**
    *   화면 우측 하단 고정 둥근 버튼 `+` 모양
    *   클릭 시 👉 "새로운 책 등록 화면"으로 이동
*   **[Bottom Navigation]**
    *   하단 탭 4개: 🏠 홈 | 💬 채팅 | ❤️ 찜목록 | 👤 마이페이지

## 3. 📖 책 상세 조회 화면 (Book Detail)

특정 도서 게시글을 클릭했을 때 나타나는 상세 정보 및 거래 액션 화면입니다.

*   **[Top Bar]**
    *   좌측: `<` 뒤로가기 버튼
    *   우측: 🔗 공유하기, 🚨 신고하기 (케밥 메뉴 `⋮` 클릭 시 노출)
*   **[이미지 캐러셀 (Image Carousel)]**
    *   좌우 스와이프 가능한 도서 이미지 목록
    *   하단 인디케이터 (예: `1 / 3`)
*   **[판매자 정보 (Profile Bar)]**
    *   판매자 프로필 이미지, 닉네임, 매너 온도 (예: 🌡️ 37.2°C)
    *   클릭 시 판매자의 다른 판매 물품 페이지로 이동 가능
*   **[핵심 정보 영역]**
    *   책 제목, 카테고리 표기 (`#전공-소프트웨어공학과`)
    *   가격 (강조)
    *   거래 상태 뱃지 ([판매중] / [예약중] / [거래완료]) 
*   **[상세 정보 테이블/그리드]**
    *   강의/교수 정보: `알고리즘` / `김폴리 교수 바뀐`
    *   책 상태: `A급` | 필기 여부: `필기 없음`
    *   거래 희망 장소: `1기술관 입구` (지도 아이콘 병기)
*   **[상세 본문 (Description)]**
    *   작성자가 입력한 본문 텍스트 표기
*   **[Bottom Action Bar (고정)]**
    *   **내가 판매자일 때:**
        *   좌측: 거래 상태 변경 드롭다운 (판매중 / 예약중 / 거래완료)
        *   우측 버튼: [수정하기]
    *   **내가 구매자일 때:**
        *   좌측: ❤️ 찜하기 버튼 (클릭 시 토글)
        *   우측 버튼: [안전결제로 구매하기] (기본) / [채팅하기] (서브)

## 4. 📝 새로운 책 등록 화면 (Create Post)

판매자가 중고 책을 등록하는 폼 화면입니다. (수정 화면도 동일 레이아웃 공유)

*   **[Top Bar]**
    *   좌측: `X` (취소/닫기)
    *   중앙: "내 책 팔기" 타이틀
    *   우측: [완료] (비활성화 상태이다가 필수값 다 채우면 활성화)
*   **[입력 폼 영역 (Scrollable)]**
    *   **사진 등록:** `[ 📷 0/10 ]` 아이콘 버튼 클릭하여 갤러리/카메라 열기. 첫 사진은 '대표' 뱃지 표시.
    *   **제목:** 텍스트 입력 (`글쓰기 가이드` 힌트 제공)
    *   **카테고리:** 라디오 버튼 또는 바텀 시트로 선택 (`전공` / `교양` / `자격증`)
        *   *전공 선택 시 나타나는 추가 입력 필드:* 학과, 학년, 학기 선택 (선택사항)
    *   **부가 정보 (선택):** 강의명, 교수님 성함
    *   **가격:** 숫자 입력 (원)
    *   **책 상태:** 버튼 그룹 (S급 / A급 / B급 / C급) 선택
    *   **필기 여부:** 버튼 그룹 (필기 없음 / 연필 필기 / 펜 필기)
    *   **상세 설명:** 멀티라인 텍스트 영역 (파손 부분, 판매 사유 등 자유 입력)
    *   **거래 희망 장소 선택:** 버튼 클릭 시 교내 건물 목록(Locations)이 바텀 시트로 올라와 선택 가능하도록 유도

## 5. 💬 채팅 목록 및 채팅방 화면 (Chat)

### [채팅 목록 화면]
*   **Top Bar:** "채팅" 타이틀
*   **목록 리스트 (List Item):**
    *   상대방 프로필 이미지
    *   우측 상단: 해당 채팅방에서 거래중인 책의 썸네일 (작게 작게 배치)
    *   상대방 닉네임, 마지막 메시지 내용, 마지막 발송 시간 (예: `어제`)
    *   안 읽은 메시지 개수 배지 (빨간 동그라미)

### [채팅방 상세 화면]
*   **Top Bar:**
    *   좌측: `<` (목록으로 가기)
    *   중앙: 상대방 닉네임 표기 (온도 표시)
    *   우측: 🚨 (신고하기, 차단, 채팅방 나가기 케밥 메뉴)
*   **상품 요약 바 (상단 고정 Sticky):**
    *   작은 책 썸네일, 책 제목, 가격, **현재 거래 상태** 표시. 클릭 시 상품 상세로 이동.
*   **대화 영역:**
    *   날짜 구분선 (예: `2026년 3월 6일`)
    *   메시지 말풍선: 좌측(상대방), 우측(나)
    *   *메시지 내부 하단:* 시간 표기 및 **읽음 확인 숫자 `1` (읽으면 사라짐)** 표시
    *   *시스템 메시지 (회색 작은 글씨):* "판매자가 거래 상태를 '예약중'으로 변경했습니다.", "에스크로 결제가 완료되었습니다." (이벤트성 알림)
*   **입력 바 (Bottom Fixed):**
    *   `+` 첨부 버튼 (사진 송부용)
    *   텍스트 입력창
    *   `전송` 버튼 아이콘

## 6. 👤 마이페이지 화면 (My Page)

사용자 본인의 활동 내역과 개인정보를 관리하는 공간입니다.

*   **[Top Bar]**
    *   "나의 Polbook" 타이틀, 우측 톱니바퀴 ⚙️(설정) 아이콘
*   **[프로필 요약 영역]**
    *   내 프로필 사진, 닉네임, 학번 정보 표기 (프로필 수정 ➡️)
    *   **매너 온도:** 온도계 바 그래프 시각화 (기본 36.5도 대비 현재 위치)
*   **[거래 내역 바로가기 (Grid Layout)]**
    *   `판매 내역` (판매중 / 판매완료)
    *   `구매 내역`
    *   `찜 목록` (하트 표시한 책들 모아보기)
*   **[결제 및 정산 (에스크로 관리)]**
    *   `결제/정산 내역` (내 예치금 확인 및 환불/정산 내역 조회)
*   **[기능 메뉴 리스트]**
    *   새로운 알림 설정
    *   내가 받은 받은 매너 평가 (받은 별점 리뷰 목록)
    *   공지사항 / 자주 묻는 질문
    *   고객센터 (버그 신고, 피드백)
    *   로그아웃 / 탈퇴하기

## 7. 💳 결제 정보 확인 화면 (Checkout)

외부 PG사 연동 시 직접 카드번호를 입력하는 화면을 만들 필요는 없으나, **PG 결제창 호출 직전 최종 확인 화면**이 필요합니다.

*   **[Top Bar]**
    *   좌측: `<` 뒤로가기, 중앙: "안전결제(에스크로)"
*   **[주문 상품 정보]**
    *   책 썸네일, 제목, 가격 노출
*   **[에스크로 안내 박스]**
    *   "안전거래를 위해 구매 확정 시까지 결제 대금을 Polbook이 보관합니다."
*   **[결제 수단 선택]**
    *   카드 결제, 계좌이체, 간편결제(카카오페이/토스페이 등) 
*   **[Bottom Action Bar (고정)]**
    *   `총 결제금액 N원` 표기
    *   [결제하기] 버튼 (클릭 시 외부 PG사 결제 모달 호출)
